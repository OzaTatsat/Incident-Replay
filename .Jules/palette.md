## 2024-05-17 - Icon-only buttons and Playback Controls Accessibility
**Learning:** Icon-only buttons like those for playback controls and event closing lack context for screen readers. Using `aria-label` provides this context, and applying `focus-visible` styling ensures users navigating by keyboard have clear indicators. In a group of buttons functioning like radio buttons (like the playback speed selectors), adding `role="group"`, an overall `aria-label`, and `aria-pressed` on individual buttons greatly improves keyboard and screen-reader usability.
**Action:** Always add `aria-label` to icon-only interactive elements and ensure focus states are clearly visible, particularly in complex UI like custom playback controllers.

## 2024-05-18 - Handling Text Overflow and Alignment in UI
**Learning:** Flexbox containers can easily cause horizontal overflow or element squishing when content exceeds the available width, especially with long text elements like file names, identifiers, or varying badge counts. D3 canvas elements also need explicit handling to prevent text elements from overlapping when drawn at the same coordinates.
**Action:** Use `flex-wrap` on container elements where items can naturally flow to the next line (like badges). For single-line text elements that shouldn't wrap, use `truncate` along with `min-w-0` on flex children, and set a `max-w-*` on the text element itself to ensure it truncates gracefully with an ellipsis. Use `shrink-0` on critical icons/buttons to prevent them from being compressed by the text. For D3, implement collision detection or staggering logic to position labels dynamically.
